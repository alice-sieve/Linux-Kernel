Debian releases are often referred to by their version name,
so it makes sense to include it along with the codename when
displaying information to the user.

Note that for some releases (namely 3.0, 4.0, 5.0 and 6.0)
the version number added to <name> does *not* match the value
contained in the <version> element, but rather the one used
by the Debian Project itself to refer to the release: the
corresponding <version>s and <short-id>s should arguably be
adjusted to match, but I'm not addressing that issue here.

Source: https://www.debian.org/releases/

Signed-off-by: Andrea Bolognani <abologna@xxxxxxxxxx>
