>   debian: Add new <short-id>s for several releases

This one, as it's right now, is going to backward compatibility.
What I'd like to do, if you agree, is to change the order and introduce
the short-ids you added *after* the current ones. With this change,
absolutely nothing is going to change *for* *now*.

I'm working on having support for multiple short-ids on libosinfo side
and once this work lands upstream, then your patches will actually be
used.

Does this make sense? Do you agree with the approach?
